# 데이터 전처리 가이드

공공데이터포털(data.go.kr)에서 업종별 CSV를 다운로드하고, Python 스크립트로 전처리하여 프론트엔드용 JSON을 생성하는 과정을 안내합니다.

---

## 1단계: CSV 다운로드

### 다운로드할 데이터셋 (4개)

| # | 업종 | 데이터셋명 | URL | 추출 카테고리 |
|:-:|------|-----------|-----|-------------|
| 1 | 치킨/카페/편의점 | 소상공인시장진흥공단_상가(상권)정보 | https://www.data.go.kr/data/15083033/fileData.do | chicken, cafe, convenience |
| 2 | 부동산 | 국토교통부_전국 부동산중개업 정보 | https://www.data.go.kr/data/15052099/fileData.do | realestate |
| 3 | 약국 | 전국약국표준데이터 | data.go.kr에서 "약국 표준데이터" 검색 | pharmacy |
| 4 | PC방 | 전국인터넷컴퓨터게임시설제공업표준데이터 | data.go.kr에서 "인터넷컴퓨터게임시설" 검색 | pcroom |

### 다운로드 방법

1. 위 URL 또는 data.go.kr 검색으로 데이터셋 페이지 접속
2. **"파일데이터"** 탭 클릭
3. CSV 형식 선택 → **다운로드** 클릭
4. 로그인 필요 시 공공데이터포털 계정으로 로그인
5. 다운로드된 파일을 `scripts/raw/` 디렉토리에 배치

### 상가(상권)정보 주의사항

- 다운로드 시 ZIP 파일로 받아짐 → 압축 해제하면 **시도별 17개 CSV** 파일이 나옴
- 압축 해제된 폴더째로 `scripts/raw/`에 넣으면 됨
- 폴더명에 "상가"가 포함되어야 스크립트가 자동 인식

### 부동산 주의사항

- 3개 파일 중 **"사무소정보"** (CSV)만 다운로드 (업자정보/공간정보 불필요)
- 파일명이 `AL_D171_00_YYYYMMDD.csv` 형식

### 파일 배치 예시

```
scripts/
└── raw/
    ├── 소상공인시장진흥공단_상가(상권)정보_20251231/
    │   ├── 소상공인시장진흥공단_상가(상권)정보_서울_202512.csv
    │   ├── 소상공인시장진흥공단_상가(상권)정보_부산_202512.csv
    │   └── ... (17개 시도)
    ├── AL_D171_00_20260408.csv          (부동산중개업사무소정보)
    ├── 건강_약국.csv
    └── 문화_인터넷컴퓨터게임시설제공업.csv
```

> `scripts/raw/`는 `.gitignore`에 등록되어 있어 git에 포함되지 않습니다.

## 2단계: Python 환경 설정

```bash
# 가상환경 생성 (권장)
python -m venv scripts/.venv
source scripts/.venv/bin/activate  # Windows: scripts\.venv\Scripts\activate

# 의존성 설치
pip install -r scripts/requirements.txt
```

## 3단계: 전처리 스크립트 실행

```bash
python scripts/process_data.py
```

### 스크립트 동작

1. **상가(상권)정보** (시도별 CSV):
   - `상권업종소분류명`으로 분류 (키워드 매칭 아님!)
   - "치킨" → chicken, "카페" → cafe, "편의점" → convenience
   - `시도명` + `시군구명` 컬럼으로 직접 집계 (주소 파싱 불필요)
   - 상가정보에는 현재 영업중인 업소만 수록됨

2. **부동산중개업사무소정보**:
   - `상태구분명` == "영업중" 필터
   - 도로명주소/지번주소에서 시도 + 시군구 파싱

3. **약국 / PC방**:
   - `영업상태명` == "영업/정상" 필터
   - 도로명주소/지번주소에서 시도 + 시군구 파싱

4. 시도별 > 시군구별 카운트 병합 → `src/data/district_counts.json` 출력

### 출력 형태

```json
{
  "서울특별시": {
    "강남구": {
      "chicken": 487,
      "cafe": 1823,
      "convenience": 612,
      "realestate": 1341,
      "pharmacy": 298,
      "pcroom": 89
    }
  }
}
```

## 핵심 컬럼 설명

### 상가(상권)정보

| 컬럼명 | 설명 | 사용 목적 |
|--------|------|-----------|
| `상권업종소분류명` | "치킨", "카페", "편의점" 등 247개 소분류 | 업종 분류 (정확한 분류 코드 기반) |
| `시도명` | "서울특별시", "부산광역시" 등 | 시도 집계 |
| `시군구명` | "강남구", "해운대구" 등 | 시군구 집계 |

### 부동산중개업사무소정보

| 컬럼명 | 설명 | 사용 목적 |
|--------|------|-----------|
| `상태구분명` | "영업중", "휴업" 등 | 영업중 필터 |
| `도로명주소` / `지번주소` | 주소 | 시도/시군구 파싱 |

### 약국 / PC방

| 컬럼명 | 설명 | 사용 목적 |
|--------|------|-----------|
| `영업상태명` | "영업/정상", "폐업" 등 | 영업중 필터 |
| `도로명주소` / `지번주소` | 주소 | 시도/시군구 파싱 |

> **인코딩**: 상가정보는 UTF-8, 나머지는 EUC-KR/CP949. 스크립트가 자동 감지합니다.

## 카카오 앱 등록 가이드

카카오톡 공유 기능을 사용하려면 카카오 개발자 앱 등록이 필요합니다.

### 등록 절차

1. [developers.kakao.com](https://developers.kakao.com) 로그인
2. **"내 애플리케이션"** → **"애플리케이션 추가하기"** 클릭
3. 앱 정보 입력:
   - 앱 이름: `동네전쟁`
   - 회사명: (개인)
4. 생성된 앱 클릭 → **"플랫폼"** 메뉴
5. **"Web"** 플랫폼 추가:
   - 사이트 도메인: `https://dongne-war.gamja.top`
6. **"앱 키"** 메뉴에서 **"JavaScript 키"** 복사
7. `src/App.svelte` 파일의 `KAKAO_JS_KEY` 상수에 붙여넣기

### 공유 테스트

1. 앱 등록 완료 후 `npm run dev`로 로컬 실행
2. 결과 카드에서 "카카오톡 공유" 버튼 클릭
3. 카카오톡 앱에서 공유 메시지 확인

> **주의**: 카카오 앱의 도메인과 실제 접속 도메인이 일치해야 공유가 동작합니다.
> 로컬 테스트 시 `http://localhost`도 플랫폼에 추가하세요.
